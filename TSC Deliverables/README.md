# COSAI TSC Deliverables

This folder tracks all deliverables produced by the COSAI Technical Steering
Committee (TSC), including papers, blog posts, tools, presentations, and
external publications.

> For weekly meeting planning, see the
> [TSC Meeting Planner and Tracker](../TSC%20Meeting%20Planner%20and%20Tracker/README.md).

---

## 🗂 How This Is Organized

| Feature | How It's Managed |
|---|---|
| Deliverable proposals | GitHub Issues using the Deliverable template |
| Timeline / timetable view | [`roadmap.md`](./roadmap.md) and the [GitHub Projects Roadmap board](../../projects) |
| Deliverable types | GitHub Issue labels (see below) |
| Progress tracking | GitHub Projects board — status updated on each Issue |

## 📌 Workflow

### Proposing a Deliverable
1. Go to **Issues → New Issue**
2. Select **Deliverable**
3. Fill in the template — title, type, owner, target date, and description
4. Apply the appropriate `deliverable-*` label
5. The group discusses and agrees to take it on — label changes to `in-progress`
6. Track progress via comments on the Issue and the Projects board

### Connecting Deliverables to Meetings
- Reference a deliverable Issue number (e.g. `#5`) in the meeting agenda file
  to discuss it at a meeting
- After the meeting, link any follow-up action item Issues back to the
  deliverable Issue via comments
- The Projects board shows both meeting action items and deliverables together

### Completing a Deliverable
1. Add a comment to the Issue with a link to the published deliverable
2. Close the Issue
3. Update [`roadmap.md`](./roadmap.md) to mark it 🟢 Complete
